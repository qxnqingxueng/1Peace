Section 01

Grand Challenge & Project Brief

1Peace addresses the civic accessibility gap — where dense government legislation creates an information divide between policymakers and the 33 million Malaysians affected by those decisions.

🏛️ Grand Challenge: Civic Accessibility & Governance Transparency

Malaysia's Parliament passes 60–80 bills per year. Each is written in legal language inaccessible to the average citizen. The majority of B40 and M40 households — over 16 million Malaysians — are unaware of aid programmes they qualify for, deadlines they must meet, or financial impacts they will experience. 1Peace bridges this gap using Agentic AI: from understanding a bill in plain language (Chat) to automatically registering for the associated aid programme (Action).

Genkit Flow — Policy Translation

// Firebase Genkit — policyTranslateFlow import { defineFlow, run } from '@genkit-ai/flow'; import { geminiPro } from '@genkit-ai/googleai'; import { vertexAISearch } from '@genkit-ai/vertexai'; export const policyTranslateFlow = defineFlow( { name: 'policyTranslate', inputSchema: TranslateInput, outputSchema: TranslateOutput }, async (input) => { // Step 1: RAG — retrieve relevant clauses (Vertex AI Search) const clauses = await run('retrieveClauses', () => vertexAISearch.query({ dataStoreId: 'malaysia-bills', query: input.query, topK: 5 }) ); // Step 2: LLM — generate plain-language summary (Gemini Pro) const summary = await run('generateSummary', () => geminiPro.generate({ prompt: buildTranslatePrompt(clauses, input.userProfile), config: { temperature: 0.2, maxOutputTokens: 1024 } }) ); // Step 3: Check if agent action is applicable const agentAction = resolveAgentAction(input.billId, input.userProfile); return { summary: summary.text(), clauses, agentAction, billId: input.billId }; } );

Ingestion Pipeline (daily batch)

StepToolDetail
1. Scrape Puppeteer Download new bill PDFs from parlimen.gov.my, Warta Kerajaan, and kdn.gov.my daily via Cloud Scheduler
2. Parse pdfplumber + regex Extract section numbers, headings, and clause text. Preserve section hierarchy for citation accuracy.
3. Chunk LangChain TextSplitter Split into 512-token chunks with 64-token overlap at clause boundaries — preserves legal context.
4. Embed + Index Vertex AI Search Push chunks to Vertex AI Search data store with metadata: bill_id, section_num, date, act_name.
5. Retrieve Vertex AI Search API Semantic search with optional filter by bill_id or category. Returns top-K chunks with relevance scores.

Autonomous Agent — Vertex AI Agent Builder

The Chat → Action escalation layer. The agent takes autonomous multi-step actions on government portals, always with human-in-the-loop approval.
Tool NameActionPortalApproval Required

enrolMySalam Enrol in MySalam free hospitalization insurance MySejahtera app Yes

🔒
Human-in-the-loop principle: The agent NEVER submits any government form without explicit citizen approval. Pre-filled forms are presented for review first. The agent may only read data and pre-fill — it cannot submit independently. This is enforced at the Vertex AI Agent Builder tool level, not just UI.
Section 10

API Specification

POST /api/chat Main chat endpoint — streams Gemini Flash response
Body: { message, sessionId, userProfile } · Response: SSE stream of tokens + sources + optional agentAction
POST /api/translate Policy translator — Gemini Pro + RAG
Body: { billId, clauseSearch?, userProfile } · Response: { summary, clauses[], agentAction?, sources[] }
POST /api/calculate Impact calculator — RAG + Gemini Pro + eligibility engine
Body: { billId, userProfile } · Response: { netImpact, breakdown[], qualifiedAid[], sources[] }
GET /api/eligibility/:uid Aid eligibility hub — rule-based match
Response: { eligible[], partial[], ineligible[] } — each with programme details, amount, deadline, applyUrl
POST /api/agent/prepare Agent — pre-fill form data for citizen review
Body: { actionId, uid } · Response: { formFields[], portalUrl, previewHtml, taskId }
POST /api/agent/submit Agent — execute submission after citizen approval
Body: { taskId, approvedFields, uid } · Response: { success, referenceNumber, confirmedAt }
POST /api/bills/subscribe Bill tracker — subscribe to a bill
Body: { billId, uid, channels: ['whatsapp','email','push'] }
GET /api/bills/:billId/diff Policy timeline — get version diff
Query: from=v2022&to=v2024 · Response: { clauses[], diff[], summary, impactChips[] }
