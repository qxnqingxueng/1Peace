⚙️ How This Is Built
Data Layer
Parliament Scraper
Cron job runs hourly. Scrapes parlimen.gov.my and hansard records. Diffs current stage vs. stored stage in PostgreSQL. Triggers alert queue on change.
Alerts
WhatsApp Business API
Twilio / Meta Cloud API sends templated alert messages when bill status changes. Email via SendGrid. Push via Firebase Cloud Messaging.
Queue
Redis + BullMQ
Alert jobs are queued per subscriber to avoid rate-limit issues. Handles retries and deduplication. Prevents duplicate alerts for same stage.
Integration
Linked to RAG + Calculator
When a tracked bill updates, the system auto-regenerates the plain-language summary and recalculates the user's economic impact, attaching both to the alert.

⚙️ How This Is Built
Rules Engine
Programme Rules DB
Each aid programme stored as a JSON rule object: income brackets, state restrictions, household type, vehicle type, dependants. Checked against user profile in O(n) — no LLM needed for the match itself.
LLM Role
Edge Case Reasoning
When a user's profile is ambiguous (e.g. income at boundary), the LLM reasons through partial eligibility and explains what additional info is needed. Purely explanatory — not the decision-maker.
Execution
Gemini Agent Forms
For programmes with open APIs (MySejahtera, PADU), Gemini Agent pre-fills application forms from the stored profile and presents them for user confirmation before submission.
Data Source
Manually curated + scraped
Programme catalogue manually curated from official sources (treasury.gov.my, jkm.gov.my). Updated quarterly. Scrapers check for new announcements. Community can flag stale data.

Plain-Language Change Summary

The 2024 amendment to §4A widens the eligibility threshold from RM4,500 to RM5,000 household income, meaning an additional ~180,000 households now qualify for the diesel subsidy.

It also removes the requirement for physical verification at JPJ, replacing it with PADU digital verification.

⚙️ How This Is Built
Text Extraction
Parliament PDF Parser
Each gazetted bill/amendment PDF is parsed using pdfplumber + custom regex to extract section numbers, headings, and clause text. Stored per-version in PostgreSQL with version tags.
Diffing
Myers Diff (diff-match-patch)
Google's diff-match-patch library runs character-level diffs on each clause pair. Additions highlighted green, deletions red. Clause-level diffing (not full-document) keeps it readable.
Summarization
LLM Change Summary
The diff is fed to the LLM with the prompt: "Summarize what this change means for a regular citizen in plain Malay and English." The impact chips (e.g. +180k new eligibles) are also LLM-extracted.
Integration
Linked to Tracker + Calculator
When Bill Tracker detects a new amendment, it triggers the Timeline Visualizer to generate a diff. The Calculator then re-runs impact estimates for subscribed users and attaches the diff link to the alert.
